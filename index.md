# MSEC CS Course Pathways

*Advising guide for MS in Economics and Computation students — Duke Computer Science*

> **Last updated:** 2026-06-12
> **Maintained by:** DGS, MSEC Program

The MSEC degree requires **at least 12 CS credits at the 500 level or higher** (typically four CS courses), with one course serving as a project-heavy capstone. Each pathway below is a 4-course backbone, ordered foundations → advanced. `*` marks a capstone candidate.

---

## 1. Algorithms

| Course | Role |
|---|---|
| **COMPSCI 531 or 532** Design & Analysis of Algorithms | Foundation; required prereq mindset for the rest |
| **COMPSCI 535** Algorithmic Game Theory | The signature MSEC course — auctions, mechanism design, equilibrium computation |
| **COMPSCI 632** Approximation Algorithms *or* **630** Randomized Algorithms | Theory depth; pick by faculty fit (Munagala/Panigrahi) |
| **COMPSCI 638** Graph Algorithms *(capstone candidate)* | Project-friendly; matches network/market problems |

```
Sem 1         Sem 2         Sem 3         Sem 4
─────         ─────         ─────         ─────
532     -->   535      -->  632/630  -->  638*
```

*Capstone fit: 638 or 535 (with a project-paper extension under independent study).*

---

## 2. AI / ML

| Course | Role |
|---|---|
| **COMPSCI 571D** Probabilistic Machine Learning | Bridges directly from ECON 608/703 econometrics |
| **COMPSCI 526** Data Science (Pei) | Applied data spine; connects econometric work to scalable data analysis |
| **COMPSCI 671D** Theory & Algorithms for ML | Rudin's course — interpretability/causality, MSEC-aligned |
| **COMPSCI 675D** Intro to Deep Learning *(capstone candidate)* | Project-heavy; satisfies capstone cleanly |

```
Sem 1         Sem 2         Sem 3         Sem 4
─────         ─────         ─────         ─────
571D    -->   526      -->  671D     -->  675D*
```

*Alternative: swap 675D and 526 if the student prefers 526 as the capstone.*

---

## 3. Systems

| Course | Role |
|---|---|
| **COMPSCI 516** Database Systems | The single most useful systems course for empirical economists |
| **COMPSCI 510** Operating Systems | Foundation if student lacks UG systems background |
| **COMPSCI 566** Systems for ML *or* **512** Distributed Systems | Pick by interest: ML infra vs. general distributed |
| **COMPSCI 514** Advanced Computer Networks *or* **564** Edge Computing *or* **655L** Full-Stack IoT Systems *(capstone candidate)* | Co-listed at the same level; pick one by thesis/project topic. 514 for networking foundations; 564 for distributed-edge workloads; 655L for hands-on systems-building project. |

```
Sem 1         Sem 2         Sem 3         Sem 4
─────         ─────         ─────         ─────
510     -->   516      -->  566/512  -->  514/564/655L*
```

*Advising note: students with strong UG CS background can drop 510 and substitute another 500-level systems elective.*

---

## 4. Security

| Course | Role |
|---|---|
| **COMPSCI 582** Cryptography | Foundation; prereq feel for 584 |
| **COMPSCI 584** Foundations of Blockchains | Strongest MSEC overlap — consensus, incentives, DeFi |
| **COMPSCI 581** Computer Security | Breadth course |
| **COMPSCI 586** Human-Centered Security & Privacy *(capstone candidate)* | Project-oriented; pairs with Machanavajjhala's privacy/fairness work |

```
Sem 1         Sem 2         Sem 3         Sem 4
─────         ─────         ─────         ─────
582     -->   584      -->  581      -->  586*
```

*Capstone fit: 586 is the cleanest project capstone; 584 also works under independent study with a research-paper output.*
